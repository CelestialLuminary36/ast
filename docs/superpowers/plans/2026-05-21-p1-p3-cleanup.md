## Task 2: 替换 matchGlob 为 doublestar

**Files:**
- Create: `internal/judge/rule_test.go`

```go
	}{
```

```go
// matchGlob performs glob matching with full support for ** segment expansion,

// Windows-style paths in `git status` output match POSIX-style patterns in

func matchGlob(pattern, s string) bool {

	s = strings.ReplaceAll(s, "\\", "/")

	return ok
}
```

## Task 3: 修复自定义 endpoint 拼接

**Files:**
- Modify: `internal/runner/api.go` (`buildClient`)
- Create: `internal/runner/api_endpoint_test.go`

**Why:** 当前 `buildClient` 仅裁剪 `/messages`，对 `https://api.anthropic.com/v1/messages` 这样的完整 endpoint 会留下 `/v1`，SDK 再 append `/v1/messages` 得到 `/v1/v1/messages`，请求 404。

### 设计

把"裁剪策略"明确化：anthropic-sdk-go 在 base URL 上 append 的是 `/v1/messages`，所以 base 必须**不含** `/v1/messages`。

- 输入 `https://api.anthropic.com/v1/messages` → base `https://api.anthropic.com/`
- 输入 `https://proxy.example.com/v1/messages` → base `https://proxy.example.com/`
- 输入 `https://proxy.example.com/anthropic/v1/messages` → base `https://proxy.example.com/anthropic/`
- 输入 `https://proxy.example.com/` → base `https://proxy.example.com/`（无需裁剪）

只需要把 `buildClient` 中的拼接逻辑抽出来便于单测。

- [ ] **Step 1: 写失败测试（抽出纯函数）**

```go

		{"https://api.anthropic.com/v1/messages/", "https://api.anthropic.com/"},
		{"https://proxy.example.com/v1/messages", "https://proxy.example.com/"},
		{"https://proxy.example.com/anthropic/v1/messages", "https://proxy.example.com/anthropic/"},
		{"https://proxy.example.com/", "https://proxy.example.com/"},
```

- [ ] **Step 3: 在 api.go 中新增 normalizeEndpointForBaseURL 并改造 buildClient**

```go
// normalizeEndpointForBaseURL converts a user-configured Messages API URL into
// the base URL the anthropic-sdk-go expects. The SDK appends "/v1/messages"
// internally, so we strip that suffix from the input if present and always
// guarantee a trailing slash.
func normalizeEndpointForBaseURL(endpoint string) string {
	s := strings.TrimSuffix(endpoint, "/")
	s = strings.TrimSuffix(s, "/v1/messages")
	if s == "" {
		return "/"
	}
	return s + "/"
}
```

修改 `buildClient` 中 endpoint 分支：

```go

    opts = append(opts, option.WithBaseURL(normalizeEndpointForBaseURL(r.cfg.Endpoint)))
}
```

删除原有 `base := strings.TrimSuffix(...)` 两行。

## Task 6: `[STEP N]` 步骤标签

**Files:**
- Modify: `cmd/ast/main.go` (`cmdTest` 场景循环内)

**Why:** PRD §7 期望读者能从输出里看到框架走到了哪一步。当前只打印"运行场景: X" 和 "[RESULT] PASSED/FAILED"，中间过程黑盒。补 4 个 STEP 标签即可让 STEP 1–4 与 PRD §7 的章节示意一一对应，不引入颜色/Unicode 框线。

### 设计

四个步骤：
1. `[STEP 1]` 初始化 Git 隔离沙盒
2. `[STEP 2]` 注入用户 Prompt 并驱动 Agent 执行
3. `[STEP 3]` 收集副作用进行规则审计
4. `[STEP 4]` 生成测试报告（在循环外，整体一次）

注：步骤 1 (workspace.New) 和 步骤 2 (rnr.Run) 在当前实现里已经是顺序操作，工作流不需要重构 —— 只是补打印。失败时打印 `[STEP N] FAILED: ...`，成功打印 `[STEP N] SUCCESS` 即可。

- [ ] **Step 1: 改造 cmdTest 场景循环**

在 `cmd/ast/main.go` 的 `cmdTest` 场景循环内，把现有的：

```go
fmt.Printf("运行场景: %s ...\n", sc.ID)
result, runErr := rnr.Run(ctx, *sk, sc, ws.Root)
```

改为：

```go
fmt.Printf("\n=== 场景: %s ===\n", sc.ID)
fmt.Printf("[STEP 1] 初始化 Git 隔离沙盒 ... ")
// workspace.New 实际在上面 ws, err := workspace.New("") 那行已经完成
fmt.Println("SUCCESS")

fmt.Printf("[STEP 2] 调用 Agent (%s) ... ", runnerName)
result, runErr := rnr.Run(ctx, *sk, sc, ws.Root)
ws.Cleanup()
if runErr != nil {
    fmt.Printf("FAILED: %v\n", runErr)
    rep.AddEntry(report.Entry{ ... })  // 既有逻辑保留
    continue
}
fmt.Printf("SUCCESS (耗时 %s)\n", result.Duration.Round(time.Millisecond))

fmt.Printf("[STEP 3] 规则审计 ... ")
judgement, err := jdg.Judge(result, sc)
if err != nil {
    fmt.Printf("ERROR: %v\n", err)
    rep.AddEntry(report.Entry{ ... })
    continue
}
if judgement.Passed {
    fmt.Println("PASSED")
} else {
    fmt.Printf("FAILED (%d 条规则未通过)\n", len(judgement.Errors))
    for _, e := range judgement.Errors {
        fmt.Printf("    - %s\n", e)
    }
}
```
